# 项目约定

## 目录 README 维护

服务端源码（apps/runtime、apps/authoring、packages/shared、db）每个目录有一份 README.md，写的是这个目录负责什么、每个文件做什么、上下游关系。改代码时如果新增/删除/移动了文件，或改变了模块职责与依赖关系，必须顺手更新所在目录的 README；只改函数内部实现不用动它。

README 的写法：全中文完整句子，不用箭头链速记（不写「A → B」），术语最多一两个放括号里解释，只写代码现状不写设计动机。

## UI 与设计系统

仓库的 UI 一律使用 @cb/ds 设计系统组件，样式只允许引用 `--cb-*` 语义 token。完整的 agent 规则手册（样式规则、组件清单、经验体 UI schema、禁令与验证命令）在仓库根的 AGENTS.md，改任何 UI 相关代码之前先读它。
