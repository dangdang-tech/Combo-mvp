# platform —— 平台层

这个目录放与具体业务无关的公用能力，按职责分五个子目录：`config/` 负责环境变量的加载与校验；`infra/` 负责外部资源客户端（数据库连接池、对象存储、登录服务验签、模型选择、进程内事件总线）并聚合成一个基础设施容器；`middleware/` 负责鉴权中间件；`http/` 负责路由声明工具、统一错误信封、健康检查和浏览器事件上报；`observability/` 负责链路追踪的启动与 trace 字段提取。

依赖方向是单向的：modules 层和 bootstrap 层向下引用这里，这里除了 `http/fastify.ts` 的类型声明引用了 agent 模块的类型外，不引用任何业务模块的运行代码。
