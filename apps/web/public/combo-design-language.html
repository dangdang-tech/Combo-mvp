Combo · Design Language 01

温暖地组织，
准确地行动。

Combo 的视觉不是“多彩”，而是让颜色各司其职。暖纸承载创作与叙事，暖墨承载运行与机器； 珊瑚表达身份和主行动，青绿色只说明“正在发生”。

The Contract

一种语言，
两种光线。

Light 与 Dark 不是两套品牌。层级、语义、组件状态保持一致，只改变明度关系和可读性配方。

  1. 1颜色先表达功能，再表达装饰。
  2. 2每个视图只保留一个占主导地位的行动。
  3. 3创作表面温暖克制，机器表面高对比且精确。
Semantic, not decorative

不能解释颜色职责，就不要使用颜色。

Editorial warmth

大字号衬线叙事，暖色块建立内容节奏。

Operational density

数据和控制保持紧凑、清晰、可扫描。

Machine clarity

日志、代码和运行表面始终采用深色机器层。

01 · Modes

暖纸 Light / 暖墨 Dark

不是纯白与纯黑的反转，而是同一套暖色世界在不同环境光下的明度重排。

Light · Warm Paper

把页面当作工作中的纸张

低对比暖白画布，内容卡片依靠色块深浅分层；边框比阴影更常用。

Dark · Warm Ink

让层级从墨色里浮现

02 · Color Semantics

每种颜色只有一个主要职责

色相不是栏目装饰，也不是随机轮换。它应该让用户在阅读文字前，就能判断对象的性质。

品牌brand

珊瑚：身份与编辑强调

用于品牌标记、精选内容和少量编辑性点睛。不是所有按钮的默认颜色。

行动action

深珊瑚：主行动

用于提交、创建、发布等不可忽略的下一步。一个区域通常只有一个。

选择selection

中性色：选择与类别

选中项、分类标签和导航归属使用暖灰层级，不借用警告色制造错误暗示。

实时live

青绿色：正在发生

运行中、同步中、重试中、在线与流式输出。完成后必须退出这个状态。

成功success

绿色：确定的正向结果

只用于已成功、已通过和确认增长，不把普通正数或所有趋势自动染绿。

警告warning

金色：需要注意但可继续

用于配额临界、待确认和非阻塞风险。不得用于普通分类或装饰高亮。

错误danger

红色：失败与破坏性动作

用于错误、删除和不可恢复风险。重试过程本身属于实时，不持续占用红色。

机器machine

暖黑：代码、日志与运行表面

稳定的高对比工作区，Light 与 Dark 中都保持比页面更深，形成“进入机器”的边界。

Token 成对，而不是颜色反相

04 · Typography

三种声音，三类信息

字体不是风格混搭。衬线负责叙事，无衬线负责操作，等宽字体负责机器输出和可核验的值。

Serif · Narrative

让能力被理解，而不只是被列出。

用于页面主标题、章节标题、编辑性引语。避免用于按钮、表格和密集配置。

Sans · Operations

创建组合并配置运行方式

用于正文、导航、表单、按钮与数据标签。保持紧凑但不拥挤。

Mono · Machine

RUNNING · 18.4s
3 tools connected

用于日志、代码、Token、时间戳与状态元数据，不用于长篇叙述。

05 · Geometry & Depth

用色块和线建立层级

Combo 的表面是平整而有节奏的。阴影只说明“悬浮”，不为每张卡片制造无意义的高度。

Spacing · 4 / 8 基线

组件内部 8–24，卡片 24–32，章节 64–96。

Radius · 8 / 12 / 18

控制 8，卡片 12，大型叙事色块 18；标签才用胶囊。

Border · 1px first

1px 用于结构，2px 只用于焦点、当前拖放或强状态。

Shadow · floating only

菜单、浮层和拖拽对象可使用阴影；静态内容卡片依赖色块。

--cb-shadow-float
06 · Components & States

状态先于装饰

组件的默认、悬停、焦点、禁用、运行、成功、警告和错误必须共享同一套语义关系。

行动层级

Primary 是当前最清晰的下一步；Secondary 是可逆的辅助动作；Ghost 是低干扰工具。

创建组合 保存草稿 查看详情 暂不可用

反馈层级

文字、图标和背景共同表达，不让颜色成为唯一信息通道。

数据保持中性

图表默认使用 Ink 灰阶与透明度建立序列。只有“成功/失败”已成为业务含义时，才使用绿/红。

  • 排名、占比、密度：中性灰阶
  • 实时采样：青绿色
  • 通过率、失败率：状态色，但必须配标签
07 · Motion

动效只解释变化

用动效连接前后状态，而不是让页面持续“活跃”。默认 120–180ms，复杂布局转换不超过 280ms。

微交互 · 120–160ms

按钮、标签和输入框的 hover / focus / pressed。只改变必要属性。

状态切换 · 160–220ms

载入完成、内容替换、局部展开。使用 ease-out，让结果快速抵达。

Reduced motion

尊重 prefers-reduced-motion：移除位移动画与循环动画，保留即时状态更新。

08 · Accessibility

可读性是系统约束

主题切换不能以降低辨识度为代价。无论环境光如何，信息层级、焦点和状态都必须可感知。

正文 ≥ 4.5:1

正文和小字号文字达到 WCAG AA；大字号至少 3:1。

焦点永远可见

键盘焦点使用 2px 轮廓与 3px 间距，不依赖 hover。

颜色不是唯一通道

状态同时提供文字、图标或形状，图表提供标签与图例。

系统偏好优先

默认跟随系统；显式选择持久化，但用户可随时恢复 System。

09 · Do / Don't

一眼判断是否属于 Combo

不是每个页面都要使用所有色彩。统一来自边界明确，而不是把同一组颜色平均撒满产品。

DO · 推荐

  • 让珊瑚集中在品牌时刻和主行动上。
  • 让选择、分类和普通数据保持中性。
  • 让青绿色只代表正在发生、在线或同步。
  • 让 Dark 保留暖色温，并保持机器层更深。
  • 用边框与色块建立结构，只给悬浮对象阴影。

DON'T · 避免

  • 不要为了“丰富”让卡片随机使用品牌色、青色和黄色。
  • 不要把分类标签染成警告色。
  • 不要把重试中、加载中误画成错误。
  • 不要默认将上升染绿、下降染红；先确认业务含义。
  • 不要在暗色模式中直接反相图片、阴影或语义色。
10 · Theme Contract

System / Light / Dark

产品使用三态偏好和二态解析结果。组件只关心解析后的主题与语义 token，不直接读取系统媒体查询。

命名规则

  • --cb-bg / --cb-surface-*：画布与层级
  • --cb-fg / --cb-muted / --cb-border：内容与结构
  • --cb-action-*：可操作层级
  • --cb-feedback-live-*：正在发生
  • --cb-state-success|warning|error-*：确定结果
  • --cb-surface-dark-*：机器表面
preference = "system" | "light" | "dark"
resolvedTheme = "light" | "dark"

html.dataset.themePreference = preference
html.dataset.theme = resolvedTheme
html.style.colorScheme = resolvedTheme

/* Components consume roles, never raw hex */
.primary-action {
  background: var(--cb-action-primary);
  color: var(--cb-on-action-primary);
}

/* System only listens while preference = system */
matchMedia("(prefers-color-scheme: dark)")
