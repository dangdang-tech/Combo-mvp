C Combo · Product Color System
Semantic palette / v1.0 / 2026.07.19

Color has a job

一色一职，
让颜色表达产品状态。

Combo 的颜色不是装饰层，而是一套语义系统：珊瑚色负责行动，米色与墨色负责选择，青色负责进行中，绿／黄／红只负责结果。

The contract

先判断含义，
再决定颜色。

同一个含义在创作者端与 Runtime 端必须使用同一个颜色角色。

  1. 1原始色值只定义色相，不直接进入组件。
  2. 2组件只消费 action / selection / live / state 等角色 Token。
  3. 3任何新颜色都必须先说明它表达的产品含义。
01 / DECISION MAP

六种问题，六种回答

这是产品中决定颜色的第一层。没有落入这六类的问题，默认回到中性色，而不是再发明一个强调色。

珊瑚 · Action 用户下一步要做什么？

主按钮、关键链接、焦点环。只表达“可行动”，不表达选中或状态。

米色 + 墨色 · Selection 用户选中了什么？

当前 Tab、选中行、模式切换。用结构对比，不与主操作抢夺视觉注意。

青色 · Live 系统正在做什么？

导入中、生成中、连接中、流式输出。结束后必须退出青色状态。

绿 / 黄 / 红 · Outcome 这件事结果如何？

成功、警告、失败。结果色永远不承担品牌、分类或主操作职责。

中性 · Category 它属于什么类型？

能力类型、内容分类、轻量标签。分类默认回到米色与墨色，不伪装成警告。

深墨 · Machine surface 这是系统产物或机器界面吗？

命令、代码、Artifact、运行日志。深色只在“机器表面”出现，不做普通页面底色。

02 / FOUNDATION

大多数界面，其实没有彩色

页面氛围由暖米色阶、暖墨文字和低对比边线建立。点击色块可复制色值。

03 / SEMANTIC ROLES

彩色必须有明确职责

原始色值不直接作为正文色。它们通过浅背景、深前景和状态组件组合，才获得可访问的产品表达。

BRAND#CC785C

珊瑚 · 品牌锚点

Logo 点缀、焦点环、少量品牌瞬间。产品主按钮使用更深的 Action 色确保可读性。

  • YES品牌标记、focus ring、稀缺的大面积品牌 Callout
  • NO成功、失败、运行中、选中 Tab、普通卡片装饰
ACTION#A9583E

深珊瑚 · 主操作

当前产品将原设计的 active 色提升为主行动色；白字对比度 5.06:1。

  • YES每个区域唯一主按钮、关键文本链接、决定性下一步
  • NO同一屏多个同权按钮、静态标签、选中或状态提示
LIVE#5DB8A6

青色 · 运行反馈

只回答“系统是否正在工作”。原色做点和轨迹，文字使用与墨色混合后的 live-fg。

  • YES导入中、生成中、连接中、实时输出、活动轨迹
  • NO完成态、主操作、普通链接、品牌装饰
CATEGORY#EFE9DE

中性色 · 类型分类

帮助扫读“它是什么”，但不制造状态暗示。分类标签使用 Card + Ink 的中性组合。

  • YES能力类型、内容类别、轻量标签、主题归属
  • NO警告黄、品牌珊瑚、成功绿、任何状态原色
SUCCESS#5DB872

绿色 · 成功结果

表示一个动作已经完成且结果可用。使用浅绿背景 + 深绿文字，不用亮绿色正文。

  • YES已发布、已完成、可用、校验通过、正向数据变化
  • NO运行中、已选择、推荐项、免费标签
WARNING#D4A017

金色 · 风险提醒

表达“可以继续，但需要注意”。它不是类别色，也不应制造普遍焦虑。

  • YES待确认、部分完成、即将过期、非阻断风险
  • NO能力类型、普通提示、品牌点缀、失败结果
ERROR#C64545

红色 · 错误与破坏

只用于失败、校验错误和危险操作。原色可做边线与图标，正文使用 error-fg。

  • YES失败、字段错误、删除、不可恢复、负向数据变化
  • NO运行中、待处理、普通空态、品牌强调
MACHINE#181715

深墨 · 机器表面

深色是内容模式，不是另一套主题。它标记命令、代码、Artifact 和系统输出。

  • YES命令行、运行日志、代码、Artifact、系统控制区
  • NO普通页面背景、常规表单、营销式大面积装饰

05 / GUARDRAILS

使用边界，比色值更重要

当一个页面看起来“颜色很多但没有重点”，通常不是色板错了，而是职责越界了。

应该这样用

  • 让 Canvas / Surface / Ink 构成页面 绝大多数视觉面积。
  • 一个操作区域只保留一个珊瑚色主行动。
  • 选中态使用米色层级与墨色标记，不抢主按钮颜色。
  • 运行中使用青色；动作结束后切换为绿、黄或红。
  • 分类、普通数据和热力强度默认使用中性色，不借用结果色。
  • 浅色状态底配深色语义文字，原始亮色只做点、线、图标。

不要这样用

  • 不要因为“这里需要一点颜色”就随机使用珊瑚、青或绿。
  • 不要让珊瑚色同时代表主按钮、当前 Tab、成功和错误。
  • 不要给分类标签上黄；黄系在产品里只保留给需要关注的 Warning。
  • 不要直接用 teal / green / amber 原色承载小号正文。
  • 不要把深墨面板扩张成全站深色主题；它只标记机器表面。
06 / CURRENT DRIFT

当前代码还要对齐的三处

这些不是换色值，而是让现有组件回到正确角色。色卡定义目标；本轮不直接改动产品结构与业务逻辑。

Category → Neutral 分类标签不再用 Amber

当前 --cb-category-* 与 Warning 同属黄系。目标改为 Surface Card + Ink；Amber 仅作为保留的编辑性点缀，不进入产品状态。

Data → Neutral 涨跌与置信度不等于成败

当前 data up/down 与 high confidence 借用了 success/error。默认应使用中性强度；只有业务明确判定 positive / negative 才映射绿红。

Retrying → Live 重试中不是错误态

当前 retrying 仍使用 error-fg。重连、重试、恢复中都属于非终态系统反馈，应统一进入 Teal / Info 角色。
