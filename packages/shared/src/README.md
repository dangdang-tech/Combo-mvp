# @cb/shared 源码总览

这个包是 authoring（创作服务）和 runtime（试用服务）两个后端共同依赖的契约层：接口的请求响应形状、错误分类、基础设施接口声明和纯函数工具都定义在这里，两个服务只引用、不重复定义。web 前端也引用其中的路由常量与视图类型。

## 文件与子目录

- `index.ts` 是包入口，把下面四个子目录的导出全部转出去。
- `core/` 是地基层，定义通用 ID、成功响应包络、游标分页、对外错误信封、任务进度模型、SSE（服务端事件推送）帧协议、健康检查契约和 traceId 工具。
- `constants/` 定义 API 路由前缀、健康探针路径和 SSE 端点路径模板。
- `ports/` 声明基础设施接口（队列、事件流、分布式锁、对象存储、大模型网关），由各服务的 infra 代码提供实现。
- `domains/` 按业务域定义数据传输对象与校验规则，覆盖登录、任务、能力项、试用会话，另含一个纯函数的去敏规则引擎。

## 约定

每个数据传输对象同时导出 `XxxSchema`（zod 运行时校验的真源）和 `Xxx`（由 schema 推导出的 TypeScript 类型），服务端校验和前端类型共用同一份定义。

## 上下游

runtime 引用 core 的错误与 trace 工具、constants 的路由前缀、ports 的对象存储接口，以及 domains 里登录、能力项、试用三个域的类型。authoring 引用面最全：除同类用法外，还实现 ports 的全部五个接口，并在提取流水线中调用 domains 的去敏引擎。具体到文件的引用清单见各子目录 README 的「上下游」小节。
