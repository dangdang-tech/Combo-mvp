> 上手 / 命令速查见仓库根 `README.md`；接口契约真源在 `docs/contracts/`；实现级架构在 `docs/01-详细技术方案.md`。

## 2. 各包职责与依赖关系

| 包                | 名称          | 职责                                                                                                                                                                                                             | 依赖                                         |
| ----------------- | ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| `packages/shared` | `@cb/shared`  | **唯一契约真源**：DTO / zod / ErrorEnvelope / 状态机 / SSE 帧协议 / 常量 / 基础设施端口接口 / OpenAPI 3.1 生成。apps 与 db 全部 import 它，绝不另立一套。                                                        | 无内部依赖（被所有人依赖）                   |
| `apps/api`        | `@cb/api`     | Fastify 多进程骨架（api/worker/consumer/sweeper 一镜像四入口）。实现 shared 的基础设施端口；路由按契约挂全路径/方法/鉴权链/幂等 scope，handler `501` 占位。                                                      | `@cb/shared`                                 |
| `apps/web`        | `@cb/web`     | React/Vite 前端骨架。typed API client（注入幂等头、解包 ErrorEnvelope）、SSE hook（12 帧协议）、统一 Loading/Error 组件、导航外壳 + 路由。                                                                       | `@cb/shared`                                 |
| `db`              | `@cb/db`      | PostgreSQL 迁移 DDL 真源 + 幂等 runner。基表先建、跨域 FK 后置闭合。**全期 36 张表**（D2，段表采 `session_segments` 单表去重；`segment_contents`/`snapshot_segments` 拆表为未采用的备选、本期不建、不计入 36）。 | `pg`（运行期）、`@cb/shared`（仅类型，按需） |
| `infra`           | `@cb/infra`   | docker-compose 编排 + Dockerfile + nginx + Redis 双配置 + 建库/建桶脚本。无 TS。                                                                                                                                 | 镜像在内部构建其余包                         |
| `scripts`         | `@cb/scripts` | 起栈 / 迁移 / 冒烟 / openapi 导出 / 集成脚本。                                                                                                                                                                   | `@cb/shared`                                 |

构建顺序由 TypeScript 项目引用编排：**`@cb/shared` 先构建**（产出 dist + .d.ts），`@cb/api`、`@cb/web` 再构建。

依赖方向是单向的：`shared ← {api, web, db, scripts}`，无环。
分层 import 规则（`eslint.config.js`）额外约束：worker/consumer/sweeper 进程禁止 import api 的 `app`/`routes`，违反即 CI 失败。

## 3. 三条硬规则在骨架里的落点

| 硬规则         | 落点                                                                                                                                                                                                 |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 绝不裸露错误码 | api `app.ts` 统一错误/404 信封 + 路由 `501` 占位信封；web `ApiError` 只暴露 `userMessage`/`action`/`retriable`/`traceId`；ErrorState 只渲染 `userMessage` + 退路按钮，traceId 仅作小字「反馈代码」。 |
| 永不裸转圈     | web `LoadingState` 三件套（ProgressBar 带量化文案 + SubtaskChecklist 逐条点亮 + Skeleton 占位）；SSE 首帧 `state_snapshot` 全量初始化 + 心跳看门狗。                                                 |
| 已生成内容不丢 | db 冻结表 schema + outbox/dead_events/consumer_cursors；迁移只加不减；幂等 scope 表 + request_hash；BullMQ jobId 去重 + fence token 随 data 入队（Phase 3 用受保护写入 CTE）。                       |

## 6. 已知一致性备注（非阻塞）

- **`REDIS_HOT_URL` 本地默认值已统一**（原两处不一致已修）：`apps/api/src/config/env.ts`、`.env.local.example`（compose 用服务名 `redis_hot:6379`）、compose（`redis_hot` 独立服务）——本机直跑均为 `redis://localhost:6380/0`（独立端口 6380、DB /0），以 compose 形态为权威。

- **OpenAPI `paths` 当前为 0**：本期 registry 注册 schema/components，path operations 留 Phase 3 填；生成器产出合法 3.1 文档骨架。
- **四进程仅 boot**：不消费队列 / 不写库；Phase 3 接 BullMQ Worker 时遵脊柱 §11.A 受保护写入 CTE。
