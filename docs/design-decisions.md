## 一、设计系统架构

### DEC-01 · Agent-ready DS 走「代码优先」而非「Figma 优先」— 已定
astryx 其实是 dev/agent 亲和（代码为源、MCP 单向镜像进 Figma），Simple DS 才是设计师亲和；用户最初的直觉标签贴反了。agent-ready 的判定标准是「fresh agent 能否自主发现组件→读懂 API→选对 token→验证结果」，这要求源头机器可读且可验证（TS 类型 + 构建回路），Figma 做源头会 drift。

### DEC-02 · 四层架构 — 已定
Token 层（DTCG JSON）/ 组件层（开放源码 + 类型化 API）/ 上下文分发层（registry + llms.txt + AGENTS.md）/ 运行时 schema 层（经验体 mini-app 受限 UI）。Figma 只作自动同步的 review 视图，不做 source of truth。

### DEC-03 · Token 是唯一事实源，app 直接消费 — 已定
`packages/ds-tokens/tokens/*.json` 是唯一源头，构建出 `dist/tokens.css`；两个前端 app 在入口 `import '@cb/ds-tokens/tokens.css'`，删除各自 `:root` 里手工重复的 `--cb-*`（消灭手工双份 canon）。改一处全局生效。

## 二、品牌与视觉

### DEC-04 · 品牌标识是固定资产，独立于 UI 强调色 — 已定
「Combo.」字标 = Fraunces Black + Com 墨 `#1C1B19` + bo./句点 砖红 `#A73718`，**永不随 UI 强调色变**。禁止用 `var(--cb-accent)` 给字标上色；禁止用可回退的 CSS 文字冒充字标（会丢 Fraunces），必须用矢量资产。（曾犯错：换肤时把字标 coral 化，已纠正并写进 design-taste skill。）

### DEC-05 · accent 最终的红 — 待定
候选：① 砖红 `#A73718`（官方规范 node 2409:24 保留的品牌红，沉稳但曾被嫌「土」）；② 珊瑚红 `#F15F43`（更亮更活，但白字对比度仅够大文本、且偏离锁定品牌）；③ 提亮版砖红（折中，保品牌+去陶土感）。目前上传/消费页暂用**砖红**。定了最终红后一个 token 值全线统一。

### DEC-06 · 画布去绿转暖 — 已定
官方冷纸底偏绿（sage 派生），按面板反馈把画布暖化：`--cb-bg #f5f3ef` / `--cb-surface #faf8f4` / `--cb-muted-bg #eeeae3` / 线条转暖。保持中性但去掉冷绿感、偏 warm-paper。

### DEC-07 · 砖红是图章不是涂料 — 已定
accent 面积预算：一屏至多 2 处强实心砖红。禁渐变、禁 emoji 当图标、禁彩虹语义色、禁字号自由值（只走 token 阶梯）。详见 `.claude/skills/design-taste`。

## 三、MVP 产品设计

### DEC-08 · MVP 导航做减法 — 已定
保留核心：工作台 / 我的能力 / 上传能力（核心动作）/ 个人主页（可分享产物）。数据分析、收益标「即将」淡化——需要流量/变现后才成立，MVP 阶段结构可见但明确延后。

### DEC-09 · 上传链路 5 步 + 软/硬 App Identity — 已定
导入对话历史 → 提取能力项 → 选择与描述 → 结构化 App Identity → 发布与试用。结构化区分：**软字段**（经验体生成、可改：name/tagline/role/goal/instructions…）+ **硬契约**（平台锁定、不随生成变：id/version/inputs schema/boundaries…）。

## 四、消费者运行时 & 伴随对话框（核心）

### DEC-10 · 消费页是运行时工作界面，不是营销页 — 已定
结构 = 侧边栏（正在运行的 mini-app 列表）+ 产品 artifact（满屏、主角）+ 伴随对话框。artifact-first，对话是伴随工具。（曾犯错：做成营销落地页，已废弃重做。）

### DEC-11 · 创作者 artifact 不约束，Combo 标准化的是中性对话框 — 已定（本条是支点）
创作者的经验体 artifact 由创作者自己渲染、样式千人千面，平台**不做约束**。Combo 真正拥有并要标准化的，是那个**中性的伴随对话框**——它的状态、位置、切换关系。设计精力应集中在对话框系统，而非某个具体 artifact。

### DEC-12 · 对话框 = 位置态 × 对话态，两维正交 — 已定
- **位置态（窗口在哪，用户控制、会记住）**：停靠·右（默认）/ 停靠·左 / 浮动 / 收纳（胶囊）/ 全屏。
- **对话态（对话在干嘛，系统驱动）**：空 / 生成中（含运行工具）/ 需补全 / 完成 / 追问 / 出错。
- 两维**正交**：任何位置都能处在任何对话态；换位置不改状态，换状态不改位置。只需分别定义两套，而非 N 种页面。

### DEC-13 · 对话框保持视觉中性 — 已定
灰阶为主，砖红只落在 发送钮 / 进行中 / 身份点；用户气泡用中性 muted-bg（非砖红软底）。理由：artifact 千人千面，对话框够中性才能衬在任何 artifact 前面不打架（DEC-11 的推论）。

### DEC-14 · 停靠态外壳精简（面板反馈落地）— 已定
删掉顶栏（COMBO·…·RUNTIME）、删掉底部流水线条；工具条只留「应用名 + 作者」；侧栏改**两层树**（应用 → 会话，激活态在会话层）。

### DEC-15 · 对话框交互模仿成熟 agent 产品（Claude/Codex）— 已定（细节待定）
多轮对话 + tool-run 卡（完成 ✓ / 进行中 spinner）+ 流式打字点 + 消息操作 + 成熟 composer。存在两版待选：**A 多轮成熟版**（身份头 + 多轮 + 停止钮/命令/上下文）vs **B 简洁版**（单轮 + 单行输入）。→ 见待定项。
