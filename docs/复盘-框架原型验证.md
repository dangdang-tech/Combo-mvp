# 复盘 · 经验体框架的原型验证(2026-06-09)

承接 `docs/重构-agentic-miniapp-经验体而非流水线.md`。那篇把产品从「填表→流水线→产物卡」
重构成「被某人经验 condition 的灵活 agent」。这次做了**尝试性实现 + 验证**,代码在 `experiments/`。

目的(用户原话):1. 验证 GUI 与 Engine 框架的兼容性;2. 从用户友好角度,这个流程/最终交付成不成立。

## 一、先做的产品设计微调:三个交互变体 + 对抗复审

用多 agent 工作流并行设计了三个差异化变体,各自自评两个目标,再对抗式打分:

| 变体 | stance | 形态 | 兼容性 | 交付/友好 | 致命伤 |
|---|---|---|---|---|---|
| **A 被问式** | advisor | 纯对话、无产物,问判断 | 最高(零新事件/通道) | 友好但**无产物=无验收**,价值易蒸发 | 回避了「产物能不能交付」,目标2 只做一半 |
| **B 协作式** | collaborator | 意图框→可见干活→可编辑产物→有环微调 | 低-中(唯一新增手改回流通道+并发面) | **交付最站得住**,环逼出品味检验 | 依赖①产物结构化可编辑②path 稳定;长文会塌 |
| **C 陪练式** | coach | 经验透明、每步亮 rationale | 低-中(要「回到任意早节点」重引擎能力) | 有「决策轨迹」二级产物 | 全押 rationale 不幻觉 + 精确有环,最易翻车 |

**结论:主推 B,对照 A,共享同一 engine。**
- B 验「交付与有环成不成立」(目标2 主场:只有它同时给结构化可编辑产物 + 有环)。
- A 验「同一 engine 能不能换皮」(目标1:对底座侵入最小,和 B 共用经验体/事件/createAgent,差异只在 GUI)。
- B 的甜区 collaborator 可向下降级 advisor、向上升级 delegate——stance 只是经验体的暴露开关,选 B 覆盖最大谱段。
- C 留作 B 通过后、要不要把 rationale 透明化升级为卖点的下一迭代。

## 二、建了什么

一个自包含原型(不接生产 share/relay,专测框架命题):

- `engine.mjs`——**变体无关引擎**。经验体(Letta 式 memory block)→ `compileSystemPrompt`(换 stance 只改 STANCE RULES 段)→ pi `createAgent` 常驻 Agent + 流式 → 解析产物契约 → `mergeAgentArtifact` / `patchArtifact`(locked-by-origin 仲裁)→ 权威事件集 SSE。**无任何 GUI 分支**。
- `advisor.html` / `collab.html`——两套界面,后端零差异。
- `brain-mock`(确定性)/ `brain-pi`(真 LLM),契约一致、可插拔。
- `test-engine.mjs`——29 项自动化验收。

## 三、验证结论

### 目标1:GUI↔Engine 解耦 —— ✅ 成立
- 同一 engine 实例只换 stance,驱动两个完全不同的前端;两路只消费权威事件集(engine 在 `_emit` 硬断言,不许发明新事件)。
- 经验体编译:换 stance 时 TASTE/GUARDRAILS/CASES 逐字不变,只 STANCE RULES 变 → 一份经验四 stance 复用。
- 浏览器 + 自动化双重实测通过。

### 目标2:流程/交付成立 —— ✅ 成立(有前提)
- **有环不塌**:浏览器里手改某格 pin → 触发 continue re-run → 锁定格保持 user 值、未锁定格被 agent 调整,version 2→4 单调无回退。
- **并发正确**:迟到 agent 输出不覆盖锁定格;取消后迟到完成不复活产物;跨 session 不串。
- **真 LLM 交付物确是「这个人的判断」**:advisor 用创作者口吻、引用真实 block、按案例模式选 A;collaborator 自己长出维度(把「三年视角」提成一列)、每格引真实 block、正确执行守则优先级(g1 一票否决),**零幻觉引用**。
- **前提=边界**:产物须结构化可编辑(长文 markdown 会让手改回流退化成模糊 diff);经验体须够厚(守则带优先级 + 案例带证据),否则塌回通用 GPT。

## 四、下一步该验的:经验提取那一端

这次用的是**手写**的经验体夹具。框架成立的真正风险不在 runtime(已证),而在 **能不能从真实 session 里
稳定抽出**这种「带优先级的判断守则 + 带证据的案例」。这是和 `docs/深度研究-创作者画像与能力提取.md`
对接的下一个实验:把萃取链的产物结构对齐成 `experiments/fixtures/` 那个 schema,再灌进同一个 engine 跑。

> 一句话:**「同一引擎换皮」和「有环手改不塌」两个最难命题已在真实链路跑通,交付物可追溯、可辨认。
> 框架的瓶颈正式从「形态成不成立」前移到「经验提取够不够好」。**
