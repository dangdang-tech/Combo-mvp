# 上传链路 · 2 步前端对齐实施计划

对齐《Agora · 能力上传链路 PRD（2 步精简版）》（Feishu wiki `PCMTw6693i4UUbkgvT8c9uMtnSf`）。本文是 PRD 与现有 5 步向导代码之间的施工说明。

## 一、PRD 目标（用户视角 2 步 + 1 过程态）

1. **Step 1 · 上传 sessions**：命令行配对上传（配对码即身份）、子任务清单（连接凭证/拉取索引/去敏/切分/生成快照）、**传完自动进入提取，无需手动点下一步**。
2. **过程态 · 自动提取能力**：系统后台从 sessions 提取能力，**非独立步骤、无用户操作**；本屏看实时进度（已分析 session 数 / 已发现能力数），完成后自动呈现能力列表。
3. **Step 2 · 能力页**：提取出的每条能力一张卡（名称 / 分类标签 / 一句话描述 / 来源 session 段数 / 发布状态）。三动作：**勾选**（默认全选）、**试用**（跳运行态试用页，本 PRD 不含其实现）、**一键发布**（勾选的一次性发布到市集，逐条「发布中」→「已发布」，给出已发布数 + 市集链接）。

明确砍掉：独立的「选择 / 结构化 / 发布」步骤、顶部步骤条、能力逐项编辑、封面 / 定价 / 单条发布 UI。

## 二、关键结论：后端已支持 2 步，改动以前端为主

排查三份现状地图后确认：

- 现有**批量发布** `POST /publish-batches`（item 传 `candidateId`）内部即跑 `create→structure→publish` 无连坐状态机（`pending→structuring→publishing→published/failed`），**正是 PRD 的「一键发布选中能力」**。
- `CreatePublishBatchItemSchema` 的 `cover/tiers/visibility` 已全部可选；worker 对候选项缺封面默认 `glyph`、缺价默认免费档、缺可见性默认 `public`（`publish/job.ts:373-380`）。即「无封面 / 无定价的一键发布」**后端零改动**即可跑通。
- 现有前端多选（subset/all）分支本来就是 `extract → select →（跳过结构化）→ 批量发布`。

因此 2 步对齐 = **前端重编排**（收 5 步为 2 步 + 过程态、去步骤条、去逐项编辑 / 封面 / 定价），后端不动 schema / 逻辑。导入→提取的「自动」由前端自动导航实现（提取仍在能力页进入时自建任务）。后端 `import→extract` 服务端自动接力是可选增强（docs/04 step6），本次不做、列为后续。

## 三、前端改动清单

沿用现有视觉语言（砖红 + 衬线标题 + 侧栏外壳），只重编排。

1. **`shell/routes.ts`** — `CREATE_STEPS` 收为 2 项：`import`（上传）、`capabilities`（能力）。`breadcrumbFor` 随之只出 2 段。
2. **`App.tsx`** — `/create` 子路由收为 `import` + `capabilities`（+ index 重定向到 import）；删 `extract/select/structure/publish` 路由与相应 import。
3. **新增 `pages/upload/step2-capabilities/CapabilitiesStepPage.tsx`** — 能力页，三相位：
   - `extracting`（过程态）：进入时若无 `extractJobId` 则 `POST /snapshots/:id/extract`，订阅 job SSE，复用 `CandidateAppearingCard`/`ExtractLoading` 流式浮现「已发现 N 个能力」。
   - `ready`：候选=能力卡网格（名称 / 分类 / 一句话 / 来源段数 / 状态）+ 勾选（默认全选）+ 每卡「试用」链接（→ 运行态，占位）+ 底部「一键发布」。
   - `publishing/done`：点一键发布 → `createPublishBatch({ items: selected.map(c=>({candidateId:c.id, idempotencyKey})) })`，订阅 batch job SSE，逐卡「发布中」→「已发布」（复用 `batchState`/`BatchResults` 归并）；完成显示已发布数 + 市集链接（`/a/:slug`，slug 取候选）。
4. **`ImportStepPage.tsx`** — 完成态由「手动下一步」改为**自动导航** `/create/capabilities?snapshotId=&draftId=`。
5. **`WizardShell.tsx`** — 移除 `StepBar` 渲染（PRD 去步骤条）；壳只留标题 + Outlet（能力页自带一键发布按钮，`WizardFooter` 视情保留/简化）。
6. **`wizardMachine.ts` / `WizardContext.tsx`** — 步骤集合与锚点收敛（去 select/structure/publish 相关派生与 `versionId/batchId` 中间锚）。
7. **退役**：`pages/wizard/SelectStepPage.tsx`(+selectApi)、`pages/upload/step4-structure/*`、`pages/upload/step5-publish/*` 中的页面级（SinglePublish/封面/定价）。被能力页复用的组件（`CandidateAppearingCard`/`ExtractLoading`/`candidateDisplay`/`batchState`/`BatchResults`）保留。
8. **`pages/index.tsx` / `pages/upload/index.ts`** — 导出同步。
9. **测试**：删除失效步骤的用例、为能力页补用例；保持 `pnpm -F @cb/web run test` 全绿（基线 82 文件 / 606 测试）。
