## (a) 一键 live 全栈跑法（需 Docker）

前置：装好 Docker + `docker compose`（v2）。在仓库根（本 monorepo 即仓库根）下：

```bash
cd <仓库根>          # 本 monorepo 即仓库根（原 creator-builder/ 已提升）

# 1) 准备生产栈环境变量：复制示例后，把所有密钥填成强随机值。
#    示例文件所有密钥项故意留空——compose 的 ${VAR:?} 会拦空值，start.sh 弱密钥守卫再拦 agora/minioadmin/postgres 等弱默认。
cp .env.compose.example .env
$EDITOR .env          # 填 POSTGRES_* / S3_* / LOGTO_APP_ID/SECRET/REDIRECT_URI / LOGTO_AUDIENCE（ANTHROPIC_API_KEY 可留空=LLM 降级）

# 2) 一键起全栈（固定启动序 + 等健康 + 跑业务迁移；任一步失败即止）。
./scripts/start.sh
#   顺序（硬性，由 depends_on+condition 落地，start.sh 再以 --wait 兜底）：
#   弱密钥守卫 → postgres/redis_queue/redis_hot/minio(healthy) → 建四桶 →
#   logto_db_seed → logto_alteration → logto(healthy=OIDC discovery issuer 自洽) →
#   业务迁移(migrate，失败即止) → 业务容器 api/worker/consumer/sweeper/web(healthy)

# 3) 基础冒烟（栈起后跑；五依赖就绪 + 三硬规则在协议面成立）。
./scripts/smoke.sh
#   断言：/health=ok、/ready 五依赖结构齐全且 ready=true、未知路由 404 ErrorEnvelope（无 code/无堆栈）、
#         /me 无 token 401 ErrorEnvelope、Logto OIDC discovery issuer 严格等于配置值且含 jwks_uri、Web 静态站可达。

# 4) 主链路 P0 验收 smoke（Phase 6 新增；对 live API 走主链路最小端到端探针）。
./scripts/acceptance-smoke.sh
#   A 段（无需登录）：四不变量在协议边界成立——
#     · 主链路全部写命令端点已注册且 requireRole 前置生效（无 token → 401，证伪「端点漏挂 / 绕鉴权 / 501 占位裸过」）；
#     · 主链路全部读端点已注册（无 token → 401）；
#     · 两条 SSE 流仅认同源 Cookie：无 token / Bearer / query token 都在【建流前】401（证「SSE 真流鉴权 + 禁混用来源」）；
#     · ErrorEnvelope 无 code：404/401 body 必含 userMessage+traceId、必不含 code/堆栈/SQLSTATE（D1 铁律）。
#   B 段（鉴权端到端，需真实会话 Cookie，可选）：
#     登录后从浏览器复制 cb_session，CB_SESSION_COOKIE='<值>' ./scripts/acceptance-smoke.sh
#     → 走 配对/上传→导入→...→发布，断言 SSE 真帧（state_snapshot+id+heartbeat）、防重幂等（同 key 回放同 jobId）。
#     无 Cookie 时优雅跳过并打印取 Cookie 步骤。

# 5) 拆栈。
pnpm -F @cb/infra compose:down
```

辅助命令：

```bash
pnpm -F @cb/infra compose:config     # docker compose config（静态校验编排，不起栈）
./scripts/migrate.sh --compose       # 单独跑业务迁移（compose 模式）
./scripts/migrate.sh --status        # 看迁移记账状态（本地模式，需 DATABASE_URL）
```

**无 Docker 时**：`start.sh` 会清晰报「需要 docker」并退出；`smoke.sh` 报「API 不可达，请先 start.sh」；
`acceptance-smoke.sh` 优雅报 `[need-docker] live 全栈未就绪` 并以**退出码 0**结束（不是失败，只是未就绪）。

## (b) 静态合并一致性检查结论

跨阶段整合（Phase 0–5）在无 Docker 下能做的全部静态核对，**结论：自洽，无致命不一致**。逐项如下。

### b.1 shared 类型在 api/web 一致引用（无类型漂移）

- `@cb/shared` 是脊柱单一真源：导出 `core/`（envelope / errors / jobs / sse / progress / structure-state / pagination / drafts / health）、
  `constants/`（idempotency / routes）、`ports/`（queue/redis/lock/object-store/llm-gateway）、`domains/`（auth/import/extract/structure/publish/dashboard/events）、`openapi/`。
- `apps/api` 与 `apps/web` 的 `package.json` 均声明 `"@cb/shared": "workspace:*"`；两侧 `tsconfig.json` 均 `references` → `../../packages/shared`；`pnpm-workspace.yaml` 纳管。
- 两侧**只**从 `@cb/shared` import（api 30+ 处、web 40+ 处）；grep 未发现 `ErrorEnvelope / ErrorBody / JobStatus / SSEEventType / StructureState` 在 app 内被本地重定义 → **零漂移**。
- `ErrorEnvelope` 形状（`packages/shared/src/core/errors.ts`）= `{ error: { userMessage, retriable, action, traceId, failureId?, details? } }`，**对外不含 `code`**；内部 `code` 仅随 `InternalError` 进日志，`sanitizeErrorEnvelope/Body` 白名单兜底。与 D1 铁律一致。

### b.2 契约 ↔ 实现端点齐全

- 契约 `docs/contracts/_index.md §2` 以「行」计 53（`/health`+`/ready` 合一行计 1）。按**路径**展开：`/api/v1` 下 54 条 + 健康探针 2 条 = **56 路径**。
- 实现侧 `apps/api/src/routes/index.ts` 的 `ALL_ENDPOINTS` 聚合各域端点声明（auth 4 / drafts 2 / import 9 / extract 5 / structure 7 / publish 7 / dashboard 14 / notifications 4 / jobs 2 = 54）+ `health.ts` 2 = **56 路径**。
- 逐域对账：契约 56 == 实现 56，**无缺失、无多余**。每条端点的方法 / 路径 / 鉴权前置 / 幂等 scope 与契约一致（见各 `routes/*.ts` 的 `*_ENDPOINTS` 声明）。

### b.3 迁移表数 + FK 闭合可建序

- `db/migrations/0000–0014` 共 15 个迁移文件，建表 **37 张**（矩阵口径 36 + `import_uploads`，0010 引入，FK 已接入）。
- FK 闭合：所有外键引用的目标表均在**同一迁移或更早**创建 → 序贯执行（0000→0014）不会前向引用失败。
- 环依赖由 `0009_post_alter_fk_closure.sql` 后置 `ALTER TABLE ... ADD CONSTRAINT` 打破（主环 `capabilities ↔ capability_versions`；`drafts` 跨域 FK；frozen 表 FK），12 条后置约束目标在其执行点都已存在。
- runner（`db/scripts/migrate.ts`）按 `readdirSync().sort()` 字典序应用，`schema_migrations` 记账（PK=文件名），每迁移 BEGIN/COMMIT/ROLLBACK 包裹；全程**只加不减**（无 DROP TABLE/COLUMN）→ 向后兼容。
- 特种表齐：outbox（`outbox_events`，0007，`seq` IDENTITY）、幂等（`idempotency_keys`，0001，lease_token 围栏）、Job/版本锁（`jobs` + 0012 `uq_structure_job_active_version` 部分唯一索引）、frozen（0008，9 表 FK 经 0009 闭合）。

### b.4 compose 服务 / 启动序 / 健康检查 / env 闭合

- `infra/docker-compose.yml` 解析有效（YAML safe_load 通过）：**14 服务**（postgres、redis_queue、redis_hot、minio、minio_mc、logto_db_seed、logto_alteration、logto、migrate、api、worker、consumer、sweeper、web）+ **3 命名卷**（pgdata、redis_queue_data、miniodata）。
- 启动序由 `depends_on.condition` 落地：数据/中间件 healthy → 建桶 → logto seed(completed) → logto alteration(completed) → logto(healthy) → migrate(completed) → 业务容器。与 `start.sh` 的 6 步一致。
- 健康检查：postgres `pg_isready` / redis×2 `redis-cli ping` / minio `mc ready` 原生探针；logto 断言 OIDC discovery `issuer` 严格等于 `LOGTO_ISSUER` 且含 `jwks_uri`；api `/health`（liveness，不查依赖）；web `wget /`。
- env 闭合：compose 引用的真实变量（16 个）全部在 `.env.compose.example` 声明（唯一「未声明」是注释里的 `${VAR:?}` 占位文本，非真变量）。canonical issuer 三处一致（logto ENDPOINT/oidc == API 验签 iss == healthcheck 断言 iss），全用 compose 服务名 `logto:3001`，不分裂。
- Redis 双实例策略对立（queue=noeviction+AOF / hot=allkeys-lru+无持久化），物理分实例，配置/健康检查/env 各自独立。

### b.5 四进程入口

- 一镜像四入口：`infra/Dockerfile.api` 构建单镜像，`infra/entrypoint.sh` 按 `PROCESS` 分叉到 `apps/api/dist/processes/{api,worker,consumer,sweeper}.js`，无效 `PROCESS` exit 64（不静默起错进程）。
- compose 在同镜像上设 `PROCESS=api|worker|consumer|sweeper` 起四服务；migrate 容器覆盖 entrypoint 跑 `db/scripts/migrate.ts`。四进程入口齐全、与 compose 对应。

### b.6 outbox / consumer / sweeper 接线

- outbox 写：`apps/api/src/events/outbox.ts` `emitInTx(tx, evt)` 在**业务状态变更同事务**内写 `outbox_events`，`event_id` 唯一去重（「发布成功必发事件、事件存在必发布成功」）。
- consumer：`apps/api/src/processes/consumer.ts` + `events/consumer-core.ts`，**单实例** PG advisory lock（`consumer:{name}`，持连即持锁，崩溃自动释放）；按「连续安全前缀水位」(cursor + xmin) effectively-once 推进；poison pill 分类处理（lifecycle 卡住等人工 / notify 退避→dead_events）。
- SSE 真流：`plugins/sse.ts` + `routes/_sse.ts` + `sse/event-stream.ts`。`text/event-stream` + `reply.hijack()` 长连接 + 15s 具名 heartbeat 事件；首帧 `state_snapshot`（或 Last-Event-ID 在窗内增量补帧）；事件源经 redis_hot Streams 桥接（worker XADD → api XREAD BLOCK），帧带 `id:`，靠 Redis Stream ms-seq 单调去重/回放。**真流、防重、可恢复**三者成立。
- sweeper：`processes/sweeper.ts` **单实例** redis_hot 租约锁（`sweeper:lock`，TTL 30s，10s 续租，崩溃自动释放）；做 Job 对账（围栏重投）、outbox 滞留检测、dead_events 重驱。

> 跨阶段整合**自洽**：脊柱类型→契约→实现→迁移→编排→四进程→outbox/SSE 链路无断点，**无致命不一致**。
> 唯一文档级陈旧（非阻塞、非本期范围）：`README.md`「本地开发」段写的测试数（api 61 等）早于当前；实跑总数已是 1543（见 b 节末 / 7 门 gate）。

## (d) 上线前 checklist

- [ ] **密钥**：`.env`（compose 用）所有密钥填强随机值；确认无空、无 agora/minioadmin/postgres 等弱默认（`start.sh` 守卫会拦，但人工再核一遍）。
- [ ] **canonical issuer 自洽**：`LOGTO_ENDPOINT`/`LOGTO_ISSUER`/`LOGTO_JWKS_URI` 三处一致，且 `LOGTO_ISSUER == {ENDPOINT}/oidc`；反代域名变更时同步改（否则 OIDC 校验全崩）。
- [ ] **回调 URL**：`LOGTO_REDIRECT_URI` 与 Logto application 配置一致、与对外域名一致；`LOGTO_AUDIENCE` 已填（生产无条件校 aud）。
- [ ] **起栈**：`./scripts/start.sh` 全 6 步绿（含业务迁移成功；迁移失败不起业务容器）。
- [ ] **冒烟**：`./scripts/smoke.sh` 全绿（/ready 五依赖 ready=true）。
- [ ] **验收探针**：`./scripts/acceptance-smoke.sh` A 段全绿；登录后跑 B 段（`CB_SESSION_COOKIE=…`）验鉴权端到端 + SSE 真帧 + 防重。
- [ ] **live 验收清单 (c)**：L1–L4、L7、L9 由 QA 在 live 栈逐项过；L5/L6 已由 CI integration 覆盖（或本地 `scripts/integration/*` 复跑）。
- [ ] **单实例约束**：`consumer` 与 `sweeper` 不可 `deploy.replicas > 1`（consumer PG advisory lock、sweeper redis_hot 租约锁兜底，但编排上别开多副本）。
- [ ] **Redis 分工**：确认 redis_queue（noeviction+AOF，挂数据卷）与 redis_hot（allkeys-lru，无数据卷）未被合并成一实例。
- [ ] **对象存储**：四桶（agora-raw/artifacts/exports/experience）建成（`minio_mc` 一次性容器，幂等）。
- [ ] **健康/接流**：编排/LB 以 `/ready` 判定接流量（非 `/health`，`/health` 仅 liveness）。
- [ ] **7 门 mock gate**：合并前 CI 绿（install/build/typecheck/web-test-typecheck/lint/format/test，共 1543 测）+ CI 的 integration、image-build job 绿。
- [ ] **回滚**：迁移只加不减（向后兼容），上一版镜像可直接回退；`pnpm -F @cb/infra compose:down` 拆栈不删命名卷（数据保留）。
