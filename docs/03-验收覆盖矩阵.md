## 四、三条硬规则贯穿核验结论

三条硬规则在每个耗时步骤（导入 / 提取 / 结构化 / 发布）逐一核验，**后端落地全部满足，有 code + 反向破坏守门测试**。

### ① 永不裸转圈 —— 满足

- 工作台 5 区块加载走 Skeleton/ChartSkeleton（脉动骨架非 spinner，`DashboardPage.tsx:75/91/106/125/154`）；`LoadingState` 全代码 grep 无 spinner/animate-spin。
- 导入：StreamLoading 三层（进度条 + 量化文案 + 五子任务）+ ItemStream 落库卡逐行（`ImportLoading.tsx:49-94`）；配对 expired 有出口。
- 提取：触发态骨架 + 逐个浮现 + 尾部骨架 + reconnecting 安抚条 + 看门狗 2× 心跳重连（`useSSE.ts:379-390`）+ slow_hint。
- 结构化：setup 骨架 + 软字段 generating Skeleton + partial + 慢生成三退路 SlowHint + streamFatal StreamLoading + 兜底 Skeleton。
- 发布：同步事务 + 按钮 busy 态；批量 job + SSE 进度（有失败也走到 100%）。
- 主页：整页 Skeleton（role=status/aria-busy）+ 分区重试中文案 + 翻页「加载中…」禁用态。
- usage/试用占位用 `UsagePlaceholder` 文案，不转圈、不裸 0、不破图。

### ② 绝不裸露错误码 —— 满足

- 对外信封统一无 code，只 userMessage + action（+ traceId）；前端 `ErrorState` 仅渲 userMessage，`sanitizeErrorBody` 白名单重建剥 code/stack（`ErrorState.tsx:74-90`，`useSSE.ts:74-84`）。
- 后端各域 handler `buildError(...userMessage...)`；测试反复断言「无 code / 无 500 / 无 NOT_FOUND」（`DashboardPage.test.tsx:253`、`ProfilePage.test.tsx:62`、`publish-routes`/`review-routes`/`publish-batch` lintUserMessage）。
- worker 异常归一 codedError（IMPORT_NO_CONTENT / DEPENDENCY_UNAVAILABLE / INTERNAL 等），字段失败发完整 ErrorEnvelope 无 code。

### ③ 已生成内容不丢 —— 满足

- SSE `state_snapshot` 重置 items 不清已生成 + Last-Event-ID 续传（`useSSE.ts:100-110`）。
- 草稿续传带全产物引用（snapshot/extract/version/batch）+ hydrateFromDraft 不重建任务（`WizardContext.tsx:159-171`）；后端 `current_step` 永不倒退（step_rank GREATEST 守门，`drafts-repo.ts:135-142`）+ 续传指针同事务不半落。
- 导入取消保留已落段；重导 INSERT 新快照 supersede 旧、旧不删；提取候选 + 证据同事务原子落库不留半残。
- 结构化 manifest + structure_state 双兜底续传，单字段重生成/编辑 surgical 落库不连坐，数组逐项 + 前缀权威防擦。
- 发布失败前端态不清 + 同 idemKey 重发；批量单项失败只标该项、已发布产物保留；评审拒绝事务回退保留上一 published 版；批次刷新经 `state_snapshot` 全量恢复。

> **结论**：本期未发现任何违反三条硬规则的实现缺陷。后端可靠性机制（idempotency fence 行为矩阵、dead_events 毒丸三语义、lifecycle 卡住停 cursor 不放错、consumer 续传水位）实现完整且有 mock 测试覆盖。三条硬规则的端到端 live 表现（断网真重连、关页真收通知）留 Phase6。
