# Combo 占卜经验体 · 交互原型

`oracle-flow.html` —— 一个**自包含、可交互**的消费流原型,是 Combo 伴随对话框(chat)系统的**活的交互 spec**。用一个「占卜 / 明日运势」创作者经验体作示例,把 Combo 平台侧要标准化的东西(对话框的状态、位置、加载态、与 artifact 的关系)完整演示出来。

> 单文件,内联 CSS + 原生 JS,零外链零构建。`file://` 双击即开,无需 server。

## 怎么跑

```bash
open prototype/oracle-flow.html      # macOS
# 或浏览器直接打开该文件
```

顶部有一条**「原型调试条 · 非产品 UI」**(仅原型可见,产品里不存在),用来快速跳到/触发各种状态,方便反复体验手感。

## 它演示了什么(对应决策)

- **完整消费流**:KOL 链接首屏(全屏聚焦 + Powered by Combo)→ 填生日 → 排盘三步仪式 → 自动停靠成运行时(DEC-19)→ 运势卡逐块揭晓 → 对话追问。
- **加载态完整性(DEC-38)**:happy-path(骨架/生成中/流式/tool-run/完成)+ adversity(慢/超时/失败/重试/断连重连)。「最关键的等待不可失败」。
- **对话态完整谱系(DEC-39)**:回合生命周期(乐观上屏→已送达→生成中→tool-run→逐字流式→完成+操作)+ 需补全(非阻断)+ **@指向 artifact**(点运势卡「财运」行→引用 chip→回答锚定+cross-highlight,DEC-22)+ 多轮自由对话 + 富 composer(@/斜杠/附件/语音)。
- **位置态机 + 尺寸驱动(DEC-40)**:头部控件切 停靠/浮动/收纳/全屏;浮窗可拖拽移动 + 拖边角缩放,**拉大→吸附成侧栏、缩小→收成胶囊**。
- **artifact 不阻塞 chat(DEC-41)**:首屏点收纳胶囊即可展开对话框、排盘前就能问它话;位置态在任何 artifact 阶段都可用。
- **渐进揭示 + 诚实侧栏(DEC-42)**:首次停靠后侧栏只有此一个 app + 一条会话;多 app 收件箱是「回访态」。
- **尺度归属 + chat 适配度(DEC-43)**:artifact 尺度归创作者(不约束);运行时框按视口约束(`height: min(设计高, 100dvh-24)`),**对话框输入永远钉在可视区**,artifact 在自己滚动区内滚。整体产品密度(-15%)。

完整决策见飞书:**Combo 设计决策记录**(docx `D1qEdfDQ8oV4i2xeWgycEnPbnY4`,DEC-38..43)。

## 给工程实现(往真产品迁移)

这个原型是「外壳 + 一个 artifact 示例」。迁移时**只把外壳做成组件,artifact 不进平台组件库**:

- **外壳(平台资产,决策已稳)**:侧栏 + 工具条 + 伴随对话框 + 位置态机 + 加载/对话态 → 迁成 `apps/runtime-web` 的 React 组件,样式只引用 `@cb/ds` + `--cb-*` token。
- **artifact 位置 = 沙盒 iframe**(DEC-29):创作者自由代码跑在 sandbox iframe,平台通过 postMessage 大协议约束边界;@指向靠注入的 selection SDK(DEC-22)。
- **chat 是平台常量组件**:中性、独立于 artifact 生命周期(DEC-41)、按视口适配(DEC-43)。
- 原型里的定时器假后端(setTimeout/setInterval 按 token 时长发事件)对应真实的:排盘生成、agent 流式、tool 调用——都要有 DEC-38 的慢/失败/重试分支。

## 已知边界(原型故意省略)

位置态的 dock-left 仅 runtime;附件/语音是演示态非真上传;部分 §8 横切叠加层(离线/SSE 重连/会话过期/破坏性确认弹窗)未做——这些留给真实实现。
