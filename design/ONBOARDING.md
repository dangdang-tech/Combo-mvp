# Combo 设计系统 + Chat 产品 · 开发者上手包

这套材料是 **Combo 消费端（伴随对话框 + 创作者经验体）** 的完整设计上下文，给准备把它做成真产品的你。原型、设计卡和契约全部离线可看；决策记录与总纲在飞书（链接见下）。**按下面顺序读**，约 1 小时能建立完整认知。

---

## 0. 一句话背景

Combo：**pro 创作者把真实会话/工作历史，沉淀成可复用的「经验体」（mini-app）；普通消费者从 KOL 链接进来使用它。**

- **artifact = 创作者的经验体产物**，千人千面、有主题，平台**不约束**（DEC-11）。
- **伴随对话框（chat）= 平台唯一标准化、真正拥有的资产**。整个产品的支点是它：中性、完整 agent 面、独立于 artifact、可在侧栏/浮窗/胶囊/全屏间流动。
- 你要建的，主要是**外壳（shell）+ 对话框 + 沙盒承载 artifact 的容器**，不是某个具体 artifact。

## 1. 先跑原型（建立直觉，最重要）→ `prototype/`

浏览器直接打开 `prototype/oracle-flow.html`（自包含单文件，`file://` 直开，无需构建）。这是**活的交互 spec**，比任何静态稿都准。

顶部有「原型调试条」，按它跑一遍，重点感受：
- 完整消费流：KOL 首屏、填生日、排盘三步仪式、自动停靠、运势卡逐块揭晓、对话追问
- **加载态**（调试条「故障模拟」）：慢排盘 / 排盘失败 / 流中断重连 / 工具失败 —— 慢和失败时界面怎么告诉你
- **对话态**：需补全、@指向（点运势卡「财运」行后引用并锚定回答）、多轮、富 composer
- **位置态**：头部控件切 停靠/浮动/收纳/全屏；浮窗拖边角缩放，拉大吸附成侧栏、缩小收成胶囊

细节见 `prototype/README.md`（调试条清单 + 每个态对应的决策）。

## 2. 读决策（为什么这么定）→ 飞书

全量决策 DEC-01 至 DEC-43 在飞书：[Combo 设计决策记录](https://zcndjgnt0026.feishu.cn/docx/D1qEdfDQ8oV4i2xeWgycEnPbnY4)。最该先看的几条（产品支点）：
- **DEC-11** artifact 不约束、对话框才是平台资产 —— 一切的支点
- **DEC-12** 位置态与对话态两维正交
- **DEC-38** 加载态完整性（happy 与 adversity，「最关键的等待不可失败」）
- **DEC-39** 对话态完整谱系 + **DEC-22** @指向
- **DEC-40** 位置态机可操作 + 尺寸驱动吸附
- **DEC-41** artifact 状态不阻塞 chat 状态（解耦）
- **DEC-42** 渐进揭示 + 侧栏是经验体收件箱
- **DEC-43** 尺度归属:artifact 归创作者，平台只保证 chat 适配度
- **DEC-29** artifact 协议 = 模型 B（沙盒 iframe + postMessage）—— 你要实现的地基

## 4. 设计系统契约 → `system/`

- `tokens/`：**token 源契约快照**：`primitives.json`（原料层裸值）+ `semantic.json`（语义层纯引用）+ `tokens.css`（构建产物，直接消费）。样式**只准引用 `--cb-*` 语义 token**。当前是密度定稿值（字号 11/12/13/14/18/24/32、间距 4/8/10/14/20/26/32/40、圆角 control6/card7/hero18）。
- `AGENTS.md`：agent 规则手册：样式规则、组件清单、经验体 UI schema、禁令、验证命令。写任何 UI 前必读。
- `design-taste.md`：品味守则：纸感/墨字/砖红图章/编辑部字体/轻；硬禁令逐条（禁渐变、禁 emoji 图标、禁彩虹语义色、禁字号自由值、禁大面积砖红等）。写 UI 和 review 前对照。

## 你要建什么（工程方向）

原型 = 外壳 + 一个 artifact 示例。迁移原则：**只把外壳做成组件，artifact 不进平台组件库。**

1. **外壳做成 React 组件**（本仓 `apps/runtime-web`）：侧栏 + 工具条 + **伴随对话框 + 位置态机 + 全部加载/对话态**，样式引用 `@cb/ds` 与 `--cb-*` token。
2. **artifact 位置 = 沙盒 iframe**（DEC-29）：创作者自由代码跑 sandbox iframe + CSP + 默认断网；平台通过 postMessage「大协议」约束边界（init/事件流/能力声明）；@指向靠注入 selection SDK（DEC-22）。
3. **chat = 平台常量组件**：中性（DEC-16）、独立于 artifact（DEC-41）、视口适配（DEC-43）、加载态完整（DEC-38）。
4. 原型里的定时器假后端，对应真实的排盘生成 / agent 流式 / tool 调用，都要带慢/失败/重试分支。

**⚠️ 唯一地基待拍板：P0-1 artifact 沙盒协议**（模型 B 已定，但消息类型/能力声明/selection SDK 协议草案未出）。建议协议定稿后再全量开工外壳与 artifact 的通信部分；外壳和对话框本身可先建。
