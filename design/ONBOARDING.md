# Combo 设计系统 + Chat 产品 · 开发者上手包

这套材料是 **Combo（伴随对话框 + 创作者能力）** 的完整设计上下文，给准备把它做成真产品的你。原型、设计卡和契约全部离线可看；决策记录与总纲在飞书（链接见下）。**按下面顺序读**，约 1 小时能建立完整认知。

> **本版重点（2026-07-13 对账更新）**：这一版做过一次**设计稿与实装（combo-mvp）的全面对账**，消除了两边打架的地方。三条你必须先知道的：
> 1. **名词统一为「能力 / 能力项」**（capability）。旧词「经验体」**已退役**，本目录内全部改完。⚠️ 仓内其他地方还有残留待同步：`ds/agent-ready-v1` 分支的仓根 AGENTS.md、CLAUDE.md 与 design-taste skill 里仍写着「经验体」。
> 2. **创作侧设计卡 = 实装镜像**，不是愿景。照 `apps/web` 真实代码重画（见第 5 节）。旧愿景（工作台 Dashboard、线性五步向导、App Identity 编辑器）**实装里没有**，已归档，别照着建。
> 3. **消费侧设计卡 = 目标 spec**，实装还没追上（见第 6 节「实装待修清单」）。这边以设计和原型为准。

## 0. 一句话背景

Combo：**pro 创作者把真实会话/工作历史，沉淀成可复用的「能力」（mini-app）；普通消费者从 KOL 链接进来使用它。**

- **artifact = 创作者的能力产物**，千人千面、有主题，平台**不约束**（DEC-11）。
- **伴随对话框（chat）= 平台唯一标准化、真正拥有的资产**。整个产品的支点是它：中性、完整 agent 面、独立于 artifact、可在侧栏/浮窗/胶囊/全屏间流动。
- 你要建的，主要是**外壳（shell）+ 对话框 + 沙盒承载 artifact 的容器**，不是某个具体 artifact。

## 1. 先跑原型（建立直觉，最重要）→ `prototype/`

浏览器直接打开 `prototype/oracle-flow.html`（自包含单文件，`file://` 直开，无需构建）。这是**活的交互 spec**，比任何静态稿都准。

顶部有「原型调试条」，按它跑一遍，重点感受：
- 完整消费流：KOL 首屏、填生日、排盘三步仪式、自动停靠、运势卡逐块揭晓、对话追问
- **加载态**（调试条「故障模拟」）：慢排盘 / 排盘失败 / 流中断重连 / 工具失败 —— 慢和失败时界面怎么告诉你
- **对话态**：需补全、@指向（点运势卡「财运」行后引用并锚定回答）、多轮、富 composer
- **位置态**：头部控件切 停靠/浮动/收纳/全屏；浮窗拖边角缩放，拉大吸附成侧栏、缩小收成胶囊

细节见 `prototype/README.md`。**注意：原型演示的位置态机、@指向、先聊后做，实装目前都还没有**——它们是目标，见第 6 节。

## 2. 读决策（为什么这么定）→ 飞书

## 4. 设计系统契约 → `system/`

- `tokens/`：**token 源契约快照**：`primitives.json`（原料层裸值）+ `semantic.json`（语义层纯引用）+ `tokens.css`（构建产物，直接消费）。样式**只准引用 `--cb-*` 语义 token**。密度定稿值：字号 **11/12/13/14/18/24/32**、间距 **4/8/10/14/20/26/32/40**、圆角 **control 6 / card 7 / hero 18**。
- `design-taste.md`：品味守则：纸感/墨字/砖红图章/编辑部字体/轻；硬禁令逐条。写 UI 和 review 前对照。**本版已修正守则里三处写错的阶梯**（旧文写着字号 16/20/28/40、圆角 8/22、旧间距，会把正确值误判成违规）。
- `AGENTS.md`：agent 规则手册：样式规则、组件清单、能力 UI schema、禁令、验证命令。

**好消息：`packages/ds` 组件库与 token 层已经和设计高度一致**（45 个文件消费 `--cb-*`、零硬编码 hex），组件实现是设计卡的**严格超集**。底座不用返工。

## 6. ⚠️ 实装待修清单（设计对、实装还没追上）

对账结论：**token 和组件底座已对齐；创作侧已用设计镜像实装；消费侧实装大面积落后于设计**。按性价比排：

**低成本、马上能改：**
- **字标换内联 SVG + 砖红锁死 `#A73718`**（两端现在用 Fraunces 900 文本 + `var(--cb-accent)`，违反守则两条铁律，换肤/离线会走样）
- **消费端用户气泡改中性 `--cb-muted-bg`**（现在是实心砖红，违反「chat 中性、颜色留给 artifact」这条承重决策，改一个 token 即修）
- **删消费端 40px 假顶栏和 mac 红绿灯**（设计刻意无顶栏、品牌退居侧栏）
- **字号收敛到 `var(--cb-text-*)`**（两个 `styles.css` 对字号 token 引用为 0，全裸 px 且有 15/17/22/26/30/34/40 等阶梯外值）；反色文字换 `--cb-accent-contrast`（现在 18 处裸 `#fff`）
- 侧栏 active 态补 accent-soft 底和左侧 3px 指示条；emoji 图标换内联 SVG；profile 封面去渐变
- 接上 `slow_hint` 帧（协议里已有，没消费）

**需排期（承重）：**
- **位置态机（DEC-40）**：实装只有浮窗一种，缺默认 dock-right、dock-left、收纳胶囊、全屏、尺寸驱动吸附
- **chat 独立于 artifact（DEC-41）**：实装把 FloatingChat 门控在首个产物之后（`hasStarted && !showGenerating`），关死了「先聊后做」
- **@指向引用（DEC-22）**：实装 iframe 只有 srcDoc 加 sandbox，无 postMessage 桥、无 selection SDK —— 这是 **Combo 的差异化 wedge**
- tool-run 可视化卡、需补全态、消息操作行（复制/重试）、富 composer

**⚠️ 唯一地基待拍板：P0-1 artifact 沙盒协议**（模型 B 已定，但消息类型/能力声明/selection SDK 协议草案未出）。建议协议定稿后再全量开工外壳与 artifact 的通信；外壳和对话框本身可先建。

## 7. 你要建什么（工程方向）

原型 = 外壳 + 一个 artifact 示例。迁移原则：**只把外壳做成组件，artifact 不进平台组件库。**

1. **外壳做成 React 组件**（本仓 `apps/runtime-web`）：侧栏 + 工具条 + **伴随对话框 + 位置态机 + 全部加载/对话态**，样式引用 `@cb/ds` 与 `--cb-*` token。
2. **artifact 位置 = 沙盒 iframe**（DEC-29）：创作者自由代码跑 sandbox iframe + CSP + 默认断网；平台通过 postMessage「大协议」约束边界；@指向靠注入 selection SDK。
3. **chat = 平台常量组件**：中性（DEC-16）、独立于 artifact（DEC-41）、视口适配（DEC-43）、加载态完整（DEC-38）。
4. 原型里的定时器假后端，对应真实的生成 / agent 流式 / tool 调用，都要带慢/失败/重试分支。
