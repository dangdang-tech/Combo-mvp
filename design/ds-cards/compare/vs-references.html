Combo vs astryx vs Simple DS

同一维度对照：定位 · token 体系 · 组件规模 · 特色层

Combo 本系统

面向「会话沉淀为能力」产品的自用设计系统：token 双层 + React 组件 + 能力 mini-app 运行时。

  • 68 个 token 变量（--cb-*）
  • 双层 DTCG：primitives → semantic
  • 20 个组件
  • 12 节点能力运行时 schema
代表页：见 Components 组实卡
Combo · 实卡即文档，不放静态截图
astryx

代码优先的 React 设计系统：文档通过 MCP 镜像给 agent 直接消费，组件即真源。

  • ~134 个变量，单层语义命名
  • 74 个组件
  • 代码优先 · MCP 文档镜像
  • 含 Motion 规范
astryx · Avatar 文档页（props 表 + 官方示例）
Simple DS

Figma 官方教学用设计系统：设计优先，变量与样式体系完备，作为团队学习范本。

  • ~347 个变量，双层 + 响应式
  • 16 个组件族
  • 设计优先 · 教学定位
Simple DS · Foundations 页（版式卡 + 色板矩阵）

astryx 取架构，Simple DS 取 token 体系，Combo 两者拼装 + 自研运行时层。
